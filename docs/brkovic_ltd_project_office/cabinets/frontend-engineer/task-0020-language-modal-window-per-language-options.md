# Task 0020 - Language Window Modal (menu + per-language variants)

**Task ID:** `BRK-MVP-FE-020`
**Owner chat:** `CHAT-BRK-FE-IMPL-001`
**Assigned:** 2026-05-29
**Status:** In Progress

## Context

На текущий момент язык выбирается из элементов меню/панели не как отдельный модальный блок. Нужно привести поведение к варианту:

- в меню сайтa остаётся только пункт `Language / Язык`;
- открытие этого пункта открывает отдельное языковое окно (модалка);
- внутри модалки показываются все доступные языки (ru, en, de, it, es, sr, zh) с единым визуальным уровнем;
- для каждого языка должен быть свой вариант текущего окна/копии/формулировки (переводы через существующий словарь).

## Required Scope

Allowed files:

- `js/main.js`
- `js/language.js`
- `css/main.css`
- `css/responsive.css`
- `admin-posts.html` / public menu templates only if explicitly needed for menu markup hooks
- `docs/brkovic_ltd_project_office/reports/**` (для отчёта по результату)

## Functional requirements

1. В общем меню/главном site-menu в нав-области оставить только пункт открытия выбора языка.
2. Удалить/скрыть прямой RU/EN переключатель в этом же контуре для этого этапа.
3. Добавить модальное окно выбора языка:
   - заголовок + список языков;
   - индикатор активного языка;
   - кнопка/действие закрытия;
   - закрытие по Esc/клику вне окна и по кнопке.
4. При выборе языка:
   - применить перевод как раньше через существующую систему языков;
   - обновить тексты UI текущего экрана;
   - обновить активную метку в пункте меню.
5. Для каждого языка отдельный контент окна/заголовков/хинтов должен браться из словаря (без привязки к ru/en только).
6. Не менять содержательную логику сайта и не запускать редизайн вне языкового блока.

## Out of Scope

- Общий SEO-переформат страниц.
- Каркасные изменения страницы/разметки других блоков.
- Backend/API.

## Acceptance Criteria

1. В меню видно ровно один пункт для перехода к выбору языка.
2. Языковое окно открывается как modal, корректно закрывается и фокус не утекает.
3. Смена языка работает на всех разрешениях для публичных страниц, где есть меню.
4. Для ru/en/de/it/es/sr/zh тексты модального окна корректны и не смешивают локали.
5. При открытии меню есть только один язык-активатор, без дублей контролов.
6. Клавиатурная и aria доступность базового модального контрола сохранена.
