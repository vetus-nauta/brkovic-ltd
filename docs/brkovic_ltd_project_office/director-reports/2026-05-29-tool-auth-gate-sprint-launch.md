## Цель спринта

Сделать аккуратный, непрерывный контур авторизации для публичного доступа к инструментам (NavDesk и т.д.) без закрытия чтения сайта:

- главные страницы и контент остаются открытыми;
- любые действия с инструментами требуют подтверждения пользователя:
  - Google login (если подключён),
  - или email OTP с 6-значным кодом;
- в UI один минимальный modal с корректным закрытием (Esc, клик по оверлею);
- все старые UX (дисплей, язык, disclaimer) без глобальной переделки.

## Важные границы

1. **Сохраняем публичный контент как сейчас.**
2. **Никаких правок дизайна страниц** без отдельного согласования: только безопасный функциональный слой.
3. **Никаких секретов в интерфейсе/логе.**
4. **Скоординированная проверка только один раз после readiness backend/FE.**

## Gate

- **BE Gate:** `GET /auth/user/me` корректно определяет сессию; защищённые tool routes возвращают auth-needed для guest.
- **FE Gate:** попытка инструмента у гостя всегда показывает auth modal; проверка не ломает чтение.
- **QA Gate:** один прогон `BRK-MVP-QAUX-013` на desktop/tablet/mobile, включая закрытие модалки и возврат к read flow.
