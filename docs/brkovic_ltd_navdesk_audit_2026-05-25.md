# BRKOVIC.LTD Nav Desk audit - 2026-05-25

Рабочая заметка перед разбором `navdesk.html` на отдельные страницы. Цель: не потерять текущие инструменты, ночной режим и дисклеймер при поэтапной переработке штурманского стола.

## Текущее состояние

Основная страница:

- `navdesk.html`
- `css/navdesk.css`
- `js/navdesk.js`

Подключены именно `css/navdesk.css` и `js/navdesk.js`.

В корне также есть старые/дублирующие файлы:

- `navdesk.css`
- `navdesk.js`

Они отличаются от подключенных файлов. До отдельной проверки их не удалять и не считать рабочими.

Текущий размер:

- `navdesk.html` - около 1300 строк
- `css/navdesk.css` - около 5700 строк
- `js/navdesk.js` - около 6900 строк

Это уже не одна страница, а набор инструментов, собранный в один экран.

## Модули внутри страницы

Крупные блоки в `navdesk.html`:

- `navdesk-watch-card` - вахтенный журнал, расписание вахт, ручной режим, текущая вахта, быстрые записи, печать/шаринг.
- `navdesk-theme-card` - переключатель день/ночь.
- `navdesk-passages-card` - расчет перехода, времени, дистанции, топлива и резерва.
- `navdesk-abbrev-card` - навигационные сокращения для чарт-плоттера и радара.
- `navdesk-tides-card` - приливы, ручной/авто режим, окно безопасного прохода.
- `navdesk-route-card` - ортодромия, локсодромия, поиск мест, промежуточные точки маршрута.
- `navdesk-ukv-card` - УКВ: радио-спеллинг, профиль судна, шаблоны VTS, marina, coast station, distress, urgency, safety.

Вывод: крупные блоки нужно выносить на отдельные страницы. На главной странице штурманского стола оставить обзор, режим экрана, дисклеймер, время, карточки инструментов и только компактные быстрые функции.

## Локальные состояния и ключи

В `js/navdesk.js` используются ключи `localStorage`:

- `navdeskAcceptedVersion` - текущий дисклеймер. Сейчас проверяется по версии, не по сроку.
- `navdesk_watch_theme_v1` - текущий day/night skin.
- `navdesk_watch_log_state_v1` - локальный черновик вахтенного журнала: основные поля вахты, быстрый черновик записи и последние записи ленты.
- `navdeskCardState:<cardId>` - состояние раскрытых/закрепленных карточек.
- `navdeskCardState:navdesk-abbrev-card` - отдельная логика для карточки сокращений.
- `navdesk_ukv_profile_v1` - сохраненный профиль судна.
- `navdesk_ukv_profile_collapsed_v2` - свернутость профиля судна после переноса УКВ на отдельную страницу. На мобильной `navdesk-ukv.html` профиль по умолчанию сворачивается в summary, но это не перетаскивается автоматически на desktop.

Нужно сохранить совместимость с этими ключами. Для дисклеймера добавить новый срок жизни около 14 дней, но не ломать пользователей, у которых уже есть старое принятие.

В handoff указано, что `navdesk.html` и `journal.html` нельзя трогать общим проходом по услугам: у них отдельная логика, ночной режим и инструменты.

Найдены материалы на Google Drive:

- `BRKOVIC.LTD Full Handoff - 2026-05-20`
- материалы для будущего раздела морского английского:
  - VHF lecture / radio phrase templates
  - COLREGs bilingual materials
  - IALA navigation marks bilingual reference
  - abbreviations glossary

Эти материалы использовать на втором этапе, когда появится отдельный инструмент обучения морскому английскому. Сейчас на главной штурманского стола нужна только карточка и заготовка страницы.

## Основные риски

- В `js/navdesk.js` много последовательных `DOMContentLoaded` блоков и исторических заплаток, особенно вокруг вахтенного журнала.
- Многие обработчики завязаны на конкретные `id`. При переносе HTML на отдельные страницы важно сохранять старые `id` внутри соответствующего инструмента.
- Ночной режим сейчас сидит в общем CSS и классе `body.navdesk-theme-night`. Новые страницы должны получать этот же класс.
- Дисклеймер сейчас модальный и блокирует страницу. Нужно перевести его на режим "показать в начале, не показывать около 14 дней после принятия", но сохранить юридический смысл.
- Мобильная версия перегружена: на маленьком экране один большой документ превращается в длинную простыню.

## Предлагаемая новая структура

Главная:

- `navdesk.html` - пульт штурманского стола.

Новые страницы инструментов:

- `navdesk-watch.html` - вахтенный журнал.
- `navdesk-tides.html` - приливы и окно прохода.
- `navdesk-route.html` - ортодромия и локсодромия.
- `navdesk-ukv.html` - УКВ, radio spelling, шаблоны сообщений.
- `navdesk-english.html` - заготовка под морской английский.

Компактные инструменты, которые можно оставить на главной:

- local / UTC time
- day/night toggle
- быстрый расчет перехода и топлива
- короткий поиск сокращений, если не перегружает экран

## Порядок безопасной работы

1. Зафиксировать текущую карту модулей и состояния - этот документ.
2. Сделать общий слой Nav Desk:
   - общий day/night init;
   - дисклеймер с TTL около 14 дней;
   - код должен работать на `navdesk.html` и будущих страницах без ошибок при отсутствии отдельных инструментов.
3. Пересобрать `navdesk.html` в главную страницу-пульт, не удаляя рабочую логику из JS.
4. Выносить крупные инструменты по одному:
   - сначала `navdesk-tides.html`;
   - затем `navdesk-route.html`;
   - затем `navdesk-ukv.html`;
   - последним `navdesk-watch.html`, потому что в нем больше всего наложенной логики.
5. После стабилизации страниц отдельно дорабатывать сами инструменты.

## Решение на ближайший шаг

Следующий технический шаг: не двигать карточки инструментов, а сначала вынести общее поведение Nav Desk в устойчивый слой внутри `js/navdesk.js`: day/night и дисклеймер с 14-дневным сроком. После этого можно начинать пересобирать главную страницу.
